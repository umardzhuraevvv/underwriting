# Системный промпт для агентов

> Этот файл — шаблон для формирования задач агентам.
> Используй его как референс при создании ТЗ.

## Обязательный порядок действий агента

1. **Прочитать CLAUDE.md** — там архитектура, модели, правила, известные грабли
2. **Прочитать ТЗ** — файл задачи из docs/tasks/
3. **Создать feature-ветку** — формат: `feature/что-делаем`
4. **Выполнить задачу** — писать код, тесты
5. **Проверить синтаксис** — `python -m py_compile файл.py`
6. **Запустить тесты** — `python -m pytest tests/ -v` (все должны быть зелёные)
7. **Закоммитить** — сообщение на русском, понятное
8. **Запушить** — `git push origin feature/название`

## Шаблон ТЗ для новых задач

```markdown
# NNN: Название задачи

- **Статус:** 📋 Запланировано
- **Дата:** YYYY-MM-DD
- **Ветка:** feature/название
- **Приоритет:** высокий/средний/низкий

## Цель
Что нужно сделать и зачем.

## Требования
1. Конкретное требование
2. Ещё одно

## Файлы для изменения
- `app/routers/anketa.py` — что менять
- `tests/test_xxx.py` — какие тесты

## Что НЕ делать
- Не ломать существующие тесты
- Не коммитить в main

## Критерий готовности
- [ ] Тесты зелёные
- [ ] Синтаксис проверен
- [ ] Коммит в feature ветку
```

## Формат команды запуска

```bash
# Вариант 1: задача текстом
./scripts/run-agent.sh "Добавить rate limiting на /api/auth/login через slowapi"

# Вариант 2: задача из файла ТЗ
./scripts/run-agent.sh docs/tasks/002_alembic.md

# Вариант 3: прямой запуск (без скрипта)
CLAUDE_DANGEROUSLY_SKIP_PERMISSIONS=true claude -p "Прочитай CLAUDE.md и docs/tasks/002_alembic.md. Выполни ТЗ."
```
