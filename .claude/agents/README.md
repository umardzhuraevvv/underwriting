# Агенты — Underwriting System

5 агентов для работы с существующим продуктом. Каждый имеет чёткую зону ответственности.

| # | Агент | Файл | Зона ответственности |
|---|-------|-------|---------------------|
| 1 | **Backend** | `backend.md` | Роутеры, сервисы, auth, бизнес-логика |
| 2 | **Frontend** | `frontend.md` | app.js, style.css, HTML-страницы |
| 3 | **Testing** | `testing.md` | pytest тесты, фикстуры |
| 4 | **Database** | `database.md` | Модели, миграции, init_db, схемы |
| 5 | **DevOps** | `devops.md` | Railway, зависимости, деплой |

## Принципы

- Каждый агент знает конкретные файлы и функции из кодовой базы
- Чёткие границы: "я меняю X, но НЕ трогаю Y"
- Единый git-воркфлоу: `dev` → `feature/*` → `dev`, никогда `main`
- Все агенты ОБЯЗАНЫ читать `CLAUDE.md` перед началом работы
